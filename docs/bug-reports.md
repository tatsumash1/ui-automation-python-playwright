# Баг-репорты

## Предисловие

Основной набор UI-автотестов для пользователя `standard_user` был выполнен успешно.

Для дополнительной проверки поведения приложения был использован пользователь `problem_user`. У этого пользователя в SauceDemo специально заложены проблемные сценарии, поэтому часть автотестов завершилась ошибкой.

Ниже оформлены дефекты, найденные по результатам падения автотестов. В каждом баг-репорте указаны шаги воспроизведения, ожидаемый результат, фактический результат и ошибка из автотеста.

## BUG-UI-001.  У problem_user кнопка Remove не удаляет товар из корзины

| Поле | Значение |
| --- | --- |
| ID | BUG-UI-001 |
| Заголовок | У `problem_user` после удаления товара счетчик корзины продолжает отображать `1` |
| Окружение | SauceDemo |
| Пользователь | `problem_user` / `secret_sauce` |
| Severity | Major |
| Priority | High |
| Предусловия | Пользователь авторизован и находится на странице товаров `/inventory.html` |
| Шаги воспроизведения | 1. Авторизоваться как `problem_user`. <br> 2. Добавить товар `Sauce Labs Backpack` в корзину. <br> 3. Проверить, что счетчик корзины показывает `1`. <br> 4. Нажать кнопку Remove у товара. <br> 5. Проверить счетчик корзины. |
| Ожидаемый результат | После нажатия кнопки `Remove` товар удаляется из корзины, кнопка снова меняется на `Add to cart`, а счетчик корзины исчезает или уменьшается. |
| Фактический результат | После нажатия `Remove` товар остается добавленным, счетчик корзины продолжает отображать 1. |
| Ошибка автотеста | `AssertionError: Locator expected to have count '0'. Actual value: 1` |
| Статус | Open |

## BUG-UI-002. Не применяется сортировка товаров Price low to high

| Поле | Значение |
| --- | --- |
| ID | BUG-UI-002 |
| Заголовок | У `problem_user` не применяется сортировка товаров по цене от низкой к высокой |
| Окружение | SauceDemo |
| Пользователь | `problem_user` / `secret_sauce` |
| Severity | Major |
| Priority | Medium |
| Предусловия | Пользователь авторизован и находится на странице товаров `/inventory.html` |
| Шаги воспроизведения | 1. Авторизоваться как `problem_user`. <br> 2. Открыть dropdown сортировки. <br> 3. Выбрать `Price low to high`. <br> 4. Проверить выбранное значение dropdown и порядок товаров. |
| Ожидаемый результат | Dropdown имеет значение `lohi`, товары отсортированы по цене от низкой к высокой. |
| Фактический результат | Dropdown остается в значении `az`, сортировка не применяется. |
| Ошибка автотеста | `AssertionError: Locator expected to have Value 'lohi'. Actual value: az` |
| Статус | Open |
