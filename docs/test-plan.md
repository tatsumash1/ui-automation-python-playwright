# Тест-план

## Цель

Проверить основные пользовательские сценарии сайта SauceDemo с помощью UI-автотестов на Python и Playwright.

## Объект тестирования

Веб-приложение SauceDemo.

URL:

```text
https://www.saucedemo.com/
```

## Инструменты

- Python
- pytest
- playwright
- pytest-playwright
- Allure Report
- Page Object Model

## Область тестирования (In-scope)

- успешный логин
- логин с невалидными данными
- добавление товара в корзину
- удаление товара из корзины
- проверка счетчика корзины
- сортировка товаров
- переход в корзину
- переход к checkout
- навигация через меню

## Что не входит в тестирование

- нагрузочное тестирование
- тестирование безопасности
- тестирование API
- кроссбраузерное тестирование на большом наборе браузеров
- мобильная адаптивность
- проверка базы данных
- платежные интеграции

## Типы тестирования

- UI-тестирование
- функциональное тестирование
- позитивное тестирование
- негативное тестирование
- регрессионное тестирование
- smoke-тестирование

## Критерии начала тестирования

- сайт SauceDemo доступен
- установлены зависимости из requirements.txt
- установлены браузеры Playwright
- настроено виртуальное окружение
- тесты запускаются через pytest

## Критерии окончания тестирования

- все запланированные автотесты выполнены
- результаты тестов зафиксированы
- Allure-отчет успешно сгенерирован
- критичных ошибок в рамках проверяемых сценариев не обнаружено

## Риски

- сайт SauceDemo может быть временно недоступен
- структура HTML-страниц может измениться
- изменение data-test атрибутов может привести к падению автотестов
- сетевые задержки могут повлиять на стабильность запуска

## Подход к автоматизации
Автотесты написаны с использованием `Page Object Model`. Логика взаимодействия со страницами вынесена в отдельные классы в папке `pages`, а тестовые сценарии находятся в папке `tests`.

## Ожидаемый результат
Все автоматизированные тесты должны проходить успешно, а результаты запуска должны отображаться в Allure Report.
